ubuntu安装多个cuda和cudnn

2025/11/6 ubuntu ubuntu cuda

cuda本身就算直接安装也不会冲突，主要是cudnn

资料来自于：

Managing multiple CUDA versions using environment modules in Ubuntu

多个CUDA版本自由切换方法 - 哔哩哔哩文章

1.安装cuda

第一个cuda可以和常规安装一样的，可以查看ubuntu安装单cuda和cudnn

这里先不要安装cudnn等会一起安装

现在假设已经安装好了cuda13.0

现在安装cuda12.5.1，这也是ubuntu24.04目前2025.11.16唯一可以拿来安装tensorflow的版本。

wget https://developer.download.nvidia.com/compute/cuda/12.5.1/local_installers/cuda_12.5.1_555.42.06_linux.run
sudo sh cuda_12.5.1_555.42.06_linux.run

然后按照下面图片来选择

这里除了CUDA Toolkit 意外全部去掉，然后进入Options

这里选择Tookit Options

然后选择Change Toolkit Install Path

复制这段路径，注意开头不要有空格，然后按enter确认并退出

回到这个页面后去掉所有选项，然后选择Done

选择Library install path (Back for system defult)

粘贴刚才复制的路径然后按enter保存并退出

选择Done

然后选择Install就可以开始安装了

至此，安装上了第二个cuda，第三第四个也是这个方法

2.安装对应的cudnn

这里的cudnn全部使用压缩包安装，压缩包安装方法参考ubuntu安装单cuda和cudnn

这里再粘一遍吧。

先去Index of /compute/cudnn/redist/cudnn/linux-x86_64下载对用的版本

这里下载9.3.0(cuda12.5)和9.16.0(cuda13.0)

这里给出直接下载的网址，点击下载

cudnn9.3.0.75 for cuda 12和cudnn9.16.0.29 for cuda 13

然后1.解压并进入文件夹 2.在终端中打开

或者也可以用指令

wegt https://developer.download.nvidia.com/compute/cudnn/redist/cudnn/linux-x86_64/cudnn-linux-x86_64-9.3.0.75_cuda12-archive.tar.xz
wegt https://developer.download.nvidia.com/compute/cudnn/redist/cudnn/linux-x86_64/cudnn-linux-x86_64-9.16.0.29_cuda13-archive.tar.xz
tar -xvf cudnn-linux-x86_64-9.3.0.75_cuda12-archive.tar.xz
tar -xvf cudnn-linux-x86_64-9.16.0.29_cuda13-archive.tar.xz

然后进入文件夹在终端中打开（这里本想全部给指令，奈何我忘了解压出的文件夹的名字，不然是可以直接cd的）

进入cudnn9.3.0然后执行

sudo cp lib/* /usr/local/cuda-12.5/lib64/ 
sudo cp include/* /usr/local/cuda-12.5/include/

再进入另一个文件夹执行

sudo cp lib/* /usr/local/cuda-13.0/lib64/ 
sudo cp include/* /usr/local/cuda-13.0/include/

至此，cuda和cudnn就安装好了

3.切换cuda

这里提供两种办法切换

一种是通过环境变量

在安装完第一个cuda后，~/.bashrc里面应该有

echo 'export PATH=/usr/local/cuda-13.0/bin${PATH:+:${PATH}}' | sudo tee -a /etc/profile.d/cuda.sh
echo 'export LD_LIBRARY_PATH=/usr/local/cuda-13.0/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}' | sudo tee -a /etc/profile.d/cuda.sh
source /etc/profile.d/cuda.sh

第二个环境变量也可以加进去变成

#export PATH=/usr/local/cuda-12.5/bin${PATH:+:${PATH}} | sudo tee -a /etc/profile.d/cuda.sh
#export LD_LIBRARY_PATH=/usr/local/cuda-12.5/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}} | sudo tee -a #/etc/profile.d/cuda.sh
#source /etc/profile.d/cuda.sh

export PATH=/usr/local/cuda-13.0/bin${PATH:+:${PATH}} | sudo tee -a /etc/profile.d/cuda.sh
export LD_LIBRARY_PATH=/usr/local/cuda-13.0/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}} | sudo tee -a /etc/profile.d/cuda.sh
source /etc/profile.d/cuda.sh

注意，这里最好注释掉其中一个，把当前不使用的全部注释掉

第二种是通过脚本

在home下新建一个switch-cuda.sh文件

就是~/switch-cuda.sh 这个路径

里面添加如下内容：

	# FITNESS FOR A PARTICULAR PURPOSE AND NONINFRINGEMENT. IN NO EVENT SHALL THE

	# AUTHORS OR COPYRIGHT HOLDERS BE LIABLE FOR ANY CLAIM, DAMAGES OR OTHER

	# LIABILITY, WHETHER IN AN ACTION OF CONTRACT, TORT OR OTHERWISE, ARISING FROM,

	# OUT OF OR IN CONNECTION WITH THE SOFTWARE OR THE USE OR OTHER DEALINGS IN THE

	set -e

	# ensure that the script has been sourced rather than just executed

	if [[ "${BASH_SOURCE[0]}" = "${0}" ]]; then

	  echo "Please use 'source' to execute switch-cuda.sh!"

	  exit 1

	fi

	INSTALL_FOLDER="/usr/local" # the location to look for CUDA installations at

	TARGET_VERSION=${1}     # the target CUDA version to switch to (if provided)

	# if no version to switch to has been provided, then just print all available CUDA installations

	if [[ -z ${TARGET_VERSION} ]]; then

	  echo "The following CUDA installations have been found (in '${INSTALL_FOLDER}'):"

	  ls -l "${INSTALL_FOLDER}" | egrep -o "cuda-[0-9]+\\.[0-9]+$" | while read -r line; do

	    echo "* ${line}"

	  done

	  set +e

	  return

	# otherwise, check whether there is an installation of the requested CUDA version

	elif [[ ! -d "${INSTALL_FOLDER}/cuda-${TARGET_VERSION}" ]]; then

	  echo "No installation of CUDA ${TARGET_VERSION} has been found!"

	  set +e

	  return

	fi

	# the path of the installation to use

	cuda_path="${INSTALL_FOLDER}/cuda-${TARGET_VERSION}"

	# filter out those CUDA entries from the PATH that are not needed anymore

	path_elements=(${PATH//:/ })

	new_path="${cuda_path}/bin"

	for p in "${path_elements[@]}"; do

	  if [[ ! ${p} =~ ^${INSTALL_FOLDER}/cuda ]]; then

	    new_path="${new_path}:${p}"

	  fi

	done	

	# filter out those CUDA entries from the LD_LIBRARY_PATH that are not needed anymore

	ld_path_elements=(${LD_LIBRARY_PATH//:/ })

	new_ld_path="${cuda_path}/lib64:${cuda_path}/extras/CUPTI/lib64"

	for p in "${ld_path_elements[@]}"; do

	  if [[ ! ${p} =~ ^${INSTALL_FOLDER}/cuda ]]; then

	    new_ld_path="${new_ld_path}:${p}"

	  fi

	done

	# update environment variables

	export CUDA_HOME="${cuda_path}"

	export CUDA_ROOT="${cuda_path}"

	export LD_LIBRARY_PATH="${new_ld_path}"

	export PATH="${new_path}"

	echo "Switched to CUDA ${TARGET_VERSION}."

	set +e

	return

脚本使用方法：

查看已经安装的cuda版本：

source switch-cuda.sh

切换到对应的版本(可在虚拟环境中使用)

source switch-cuda.sh 12.5

确认当前环境的cuda版本

nvcc -V

注意，cuda是要在环境变量里配置的，环境变量里是哪个cuda，新开的终端就是哪个cuda，本文件只对当前终端有效。
