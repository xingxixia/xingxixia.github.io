删除多系统下的ubuntu

2025/11/22 ubuntu ubuntu windows

win11，ubuntu22.04和ubuntu24.04

我的ubuntu22.04只有93G，后面紧跟着24.04，没地方扩容了，打算换个地方，但是又忘了怎么删除efi分区，所以记录一下。

我的ubuntu24.04efi分区和主分区都是单独的，删除的时候直接删除就行了，这个很好说，但是ubuntu22.04和win11的引导是在一起的，这就比较麻烦了，我们需要单独打开那块盘然后删除ubuntu的分区。

引用：

快速完全删除ubuntu，删除双系统_哔哩哔哩_bilibili

双系统下完全卸载ubuntu_哔哩哔哩_bilibili

总共有两部分需要删除，一部分是主分区，一部分是EFI分区，关于删除主分区，直接在磁盘管理里面右击然后删除卷就行，EFI分区比较麻烦，原因是它和windows是在一个分区里的，所以不能直接删除分区，所以接下来主要记录删除efi分区

打开cmd，然后输入list disk，此时会列出所有硬盘及其信息

找到安装ubuntu22.04efi文件的磁盘，我这里是磁盘1，所以再输入select disk 1

注：select也可以是sel

此时就已经选中了磁盘1，如果此时再次输入list disk的话会发现disk1的前面多了个*

输入list part会列出硬盘1上所有的分区。找到efi分区，也就是系统分区，我这里是part1

所以输入sel part 1，此时我们就选中了硬盘1下的分区1，也就是安装ubuntu22.02引导文件的地方

输入assign letter=Z这里是分配盘符，其中z也可以是其他大写字母，但注意不要和现有的盘符重复了

接下来的操作就有很多种了，

由于安全原因，不能直接打开这个文件夹，因此我们需要用其他手段进去删掉ubuntu

1 使用powershell

使用管理员身份运行powershell，然后输入cd Z:这个Z是刚在分配的盘符，冒号别忘了。

再输入ls可以看到里面有一个EFI文件夹，输入cd .\EFI\或cd EFI就能进入该文件夹。

再次输入ls就可以看到里面有个ubuntu文件夹

执行rm .\ubuntu\就能删除ubuntu的引导文件了。

2 使用其他能打开文件的软件—以记事本为例子

虽然直接去点击是打不开的，但是如果有管理员权限就能打开，在搜索栏里面输入记事本，然后使用管理员身份打开

此时我们的身份就是管理员了，选择文件–打开–z盘，你会惊喜的发现这样是可以进去的，点进去找到ubuntu文件夹删掉就好了

3 简单粗暴的登录管理员账户

题目就是方法

最后别忘了关闭z

在diskpart里面输入 remove letter=Z
